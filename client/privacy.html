Privacy Policy

Ghost Chat — zero-trace P2P encrypted messenger

1. Overview

Ghost Chat is a peer-to-peer encrypted messenger designed with privacy as its core principle. We do not collect, store, or process any personal data or message content.

2. Data We Do NOT Collect

  • Messages: All messages are end-to-end encrypted (AES-256-GCM) and exist only in device memory. They are never stored on any server.
  • User accounts: Ghost Chat does not require registration, login, email, phone number, or any personal information.
  • Metadata: We do not log who talks to whom, when, or how often.
  • IP addresses: The signaling server does not log IP addresses. Privacy mode routes all traffic through TURN relay servers to hide IP addresses from peers.
  • Device information: We do not collect device identifiers, OS version, or any device metadata.

3. How It Works

Ghost Chat uses WebRTC for direct peer-to-peer connections. The signaling server only relays WebRTC handshake data to establish the connection — it is fully stateless and retains no information after the connection is established.

  • Encryption: ECDH P-256 key exchange, HKDF key derivation, AES-256-GCM
  • Perfect Forward Secrecy: Keys rotate every 50 messages
  • Messages are padded to 256-byte blocks to prevent traffic analysis
  • Replay protection with nonce tracking and counter windows

4. Voice Calls

Voice calls are transmitted directly between peers using WebRTC with DTLS-SRTP encryption. Audio data never passes through our servers. Call signaling is encrypted end-to-end through the existing secure data channel.

5. Data Retention

There is no data retention because there is no data to retain. Messages exist only in volatile memory and are permanently lost when you close the app or leave the chat. Empty rooms are automatically deleted after 10 minutes.

6. Third-Party Services

Ghost Chat uses STUN/TURN servers for WebRTC connectivity:

  • Google STUN server (stun.l.google.com) — for NAT traversal only
  • Cloudflare STUN server (stun.cloudflare.com) — for NAT traversal only
  • Self-hosted TURN server — for relaying encrypted traffic when direct connection is not possible

No analytics, tracking, or advertising services are used.

7. Microphone & Camera Permissions

The app requests microphone access only when you initiate or accept a voice call. Camera permission is declared for WebRTC framework compatibility but is not actively used. No audio or video is recorded or stored.

8. Children's Privacy

Ghost Chat is not directed at children under 13. We do not knowingly collect any information from anyone, including children.

9. Changes to This Policy

We may update this privacy policy from time to time. Changes will be posted on this page.
